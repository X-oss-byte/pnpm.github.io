버전: Next

프로덕션

pnpm을 통해 프로덕션 환경에서 여러분의 패키지를 부트스트랩하는 두 가지 방법이 있습니다. 그 중 하나는 lockfile을 커밋하는 것입니다. 그런 다음 여러분의 프로덕션 환경에서 pnpm install 을 실행합니다. 이것은 lockfile을 사용하여 의존성 트리를 빌드합니다. 즉, 의존성 버전은 lockfile이 커밋되었을 때와 일치하게 됩니다. 이는 의존성 트리가 여러 환경에서 지속되도록 하는 가장 효과적인 방법 (또한 가장 권장하는) 입니다.

다른 방법은 lockfile을 커밋하고 패키지 스토어를 여러분의 프로덕션 환경에 복사하는 것입니다 (store location option으로 위치를 변경할 수 있습니다). 그런 다음 pnpm install --offline을 실행하여 pnpm이 전역 스토어로부터 패키지를 사용하게 되며, 레지스트리에 어떤 요청도 하지 않게 됩니다. 이 기능은 어떤 이유로든 레지스트리에 대한 외부 접근이 불가능한 환경에서만 권장됩니다.
